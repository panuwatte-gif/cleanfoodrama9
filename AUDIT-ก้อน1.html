สูตรพยากรณ์ที่เจ้าของให้ ไม่ต้องใช้ POS รายเมนู — มันอนุมานยอดขายจาก ผลต่างของการนับสต๊อกรายวัน ตรงๆ:
ขายไป(t) = คงเหลือ(t−1) + รับเข้า(t) − ของเสีย(t) − คงเหลือ(t)
ปัญหาคือตอนนี้ applyCount / applyReceive / applyWaste ใน store.js เขียนทับคงเหลือปัจจุบันอย่างเดียว ไม่เก็บสแนปช็อตรายวัน → ตาราง rama9_stock_counts / waste_logs เลย 0 แถว → สูตรไม่มี คงเหลือ(t−1) ให้คำนวณ. นี่ไม่ใช่ "ไม่มีข้อมูลเลยทำไม่ได้" — ต้นทางมีจริง (การนับทุกวัน) แค่ยังไม่ถูกบันทึกเป็นประวัติ. ก้อน 2.3 = ต่อ apply* ให้ append แถวรายวัน แล้วเขียน engine ตามสูตรเป๊ะ → วันแรกที่เริ่มนับ ระบบคำนวณยอดขาย/พยากรณ์จากข้อมูลจริงทันที (ช่วง cold-start ใช้ค่าเฉลี่ยตามสเปก ขั้น 2.5).

ขอยืนยันก่อนเริ่มก้อน 2 — ยังไม่แตะโค้ดจนกว่าจะ OK

สรุปงานที่จะลงมือ (ตามลำดับสเปก) เมื่อเจ้าของยืนยันตารางนี้:
  1. 2.1 เพลง: สร้างตาราง rama9_songs (รัน SQL ตรง) + sync รายชื่อขึ้นคลาวด์ · แปลงไฟล์เป็น MP3 ก่อนเก็บ (พิสูจน์ขนาด KB ก่อน/หลัง)
  2. 2.2 เมนู·ราคา: ตารางเดี่ยว standalone (กับข้าว · ชนิดข้าว · ราคาตั้ง · ส่วนลด · สุทธิ) ไม่มี FK — จดโปรกันลืม
  3. 2.3 พยากรณ์: ต่อ count/recv/waste ให้บันทึกสแนปช็อตรายวัน → เขียน engine ตามสูตรเป๊ะ → ปลดล็อก ขายดี/ขายน้อย/พยากรณ์ ให้เป็นจริง (cold-start ใช้ค่าเฉลี่ย)
  4. 2.4 บีบรูป: ตรวจทุกจุดอัปรูป → ย่อ ≤1200px + WebP ก่อนอัป (พิสูจน์ขนาดจริงใน Storage)
  5. ก้อน 3: กวาด demo/seed ที่เหลือ (MONEY · TOP/BOTTOM · DOW_SALES · COUNT_RESULT · INCOME_LOG · FC_HISTORY · RECORDS · SHRINK_DEMO) ออกทุกหน้า → เหลือ empty state ซื่อสัตย์
